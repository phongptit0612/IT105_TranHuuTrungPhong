Cấu trúc tài liệu SRS – Hệ thống Học Trực Tuyến

| **Mục**                                                          | **Giải thích / Vai trò**                                                                                           |
| ---------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| **1. Giới thiệu (Introduction)**                                 | Trình bày mục tiêu, phạm vi và đối tượng người dùng của hệ thống học trực tuyến.                                   |
| **2. Mô tả tổng quát (Overall Description)**                     | Tóm tắt bối cảnh, các chức năng chính, loại người dùng và ràng buộc của hệ thống.                                  |
| **3. Yêu cầu chức năng (Functional Requirements)**               | Liệt kê chi tiết các chức năng mà hệ thống phải cung cấp, ví dụ: đăng nhập, tham gia khóa học, nộp bài, chấm điểm. |
| **4. Yêu cầu phi chức năng (Non-functional Requirements)**       | Mô tả các yêu cầu về hiệu năng, bảo mật, giao diện, khả năng mở rộng, tính sẵn sàng của hệ thống.                  |
| **5. Mô hình dữ liệu và giao diện (System Models / UI Mockups)** | Trình bày sơ đồ use case, ERD hoặc bản phác thảo giao diện giúp hình dung hệ thống.                                |
| **6. Giả định và ràng buộc (Assumptions and Constraints)**       | Nêu các điều kiện tiên quyết và giới hạn về công nghệ, pháp lý hoặc nguồn lực.                                     |
| **7. Phụ lục (Appendix)**                                        | Cung cấp thông tin bổ sung như thuật ngữ, tài liệu tham khảo hoặc hướng dẫn đặc biệt.                              |
